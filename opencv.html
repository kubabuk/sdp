Installing OpenCV 2.2.0 (locally) on DICE

by s0840449 » Mon Jan 24, 2011 12:54 pm

Okay guys, since we're having such problems with OpenCV on DICE, I offered to post up my local install guide for it. This should get OpenCV up and running on your DICE account, and have it accessible from Python. We're not using C++ or anything, so if you're using a language other than Python you'll have to work out how to import/include opencv in your code on your own!

Installing OpenCV
  1. Download the OpenCV source files from http://sourceforge.net/projects/opencvl ... -unix/2.2/
  2. Untar the file to a local folder for installation. In this guide, I use "~/SDP"
  3. Change directories into the newly created OpenCV-2.2.0 folder.
    Code: Select all
    cd ~/SDP/OpenCV-2.2.0
  4. In this folder, make a new folder called "release".
    Code: Select all
    mkdir release
  5. Move into the release folder.
    Code: Select all
    cd release
  6. Run the GUI version of cmake. To get this working, I had to run cmake once, and then ccmake afterwards.
    Code: Select all
    cmake .. && ccmake ..
  7. This will bring up a configuration options screen. Here, change the following variables:
    1. CMAKE_INSTALL_PREFIX - change this to your preferred installation directory. For example, "~/SDP/opencv"
    2. WITH_FFMPEG - change this to "OFF". (You have to scroll down off the bottom of the screen to see this option.)
  8. Now run "make". You will need about 400-500MB of free space on your disk - be aware of this! If you can't afford the space, try moving the installation folder to /tmp or to the scratch space if you're on one of the SDP machines. You won't need the install folder when you're done, so its safe to have it there.
    Code: Select all
    make
  9. Make will take a while. Be patient. Hope to [your choice of deity here] it doesn't fail.
  10. Now, run "make install". Be prepared to wait for a bit again. But less time this time around.
    Code: Select all
    make install
  11. Assuming all has gone well, you are pretty much done. Next up, I'll deal with getting Python working with OpenCV.
  12. (As an aside, you can free up 400MB of space by now deleting the installation folder (OpenCV-2.0.0, not the opencv folder!))

Getting Python working with OpenCV
  • In order to use opencv with Python, we need to add the location of the OpenCV library to your python modules path. Run the following command, substituting in your own username and install directory. It MUST be a full path. You may wish to add this command to your "~/.bashrc" to stop you having to type it every time you open a new terminal.
    Code: Select all
    export PYTHONPATH=$PYTHONPATH:/afs/inf.ed.ac.uk/user/s08/s0840449/SDP/opencv/lib/python2.6/site-packages
  • That's pretty much it. To test it out just run the python interpreter:
    Code: Select all
    python
  • ... and try to import the cv library:
    Code: Select all
    import cv
  • If that succeeded then congratulations - you now have Python support! Go have fun.

Known Issues with OpenCV on DICE
All of these issues were encountered when using Python. If you find that these issues do not occur with your choice of language, please do stop by and tell everyone! Also, if you find other issues, or work-arounds for these issues, please let me know so I can fix the guide.
  • Theres no FFMPEG support. DICE has FFMPEG, but OpenCV doesn't seem to like it. It may be because we're missing the FFMPEG developer files.
  • You can't save an image as a "PNG" file. Don't ask me why - it seg faults after complaining about something being mismatched. Don't cry too hard.
  • The image is occasionally (read "often") interlaced. But not always. Again; who knows?
  • Grabbing a frame from the camera seems to return an image that is bigger than the actual camera size, so there's a horrible bit of black noise to the left of the image. We're not sure if this is actually present in the image taken from the camera, or if it just appears when either using cv.ShowImage or cv.SaveImage...


Hope that helps someone,

A little info on two of the known issues.

The black lines down the side issue is a known one. Its not DiCE related its simply a case
of more information being captured than what is being transmitted. Cropping the image
is the solution to this one.

The interlacing is almost certainly down to the fact that the video signal format is PAL I S-Video
(625 lines at 25 frames per second) which is an interlaced signal.
